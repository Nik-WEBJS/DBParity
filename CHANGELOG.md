# Changelog

## 0.7.0 — 2026-07-03

- **feat(cli)**: `dbparity history` — таймлайн дрейфа dual-write: журнал
  инкрементальных прогонов в стейте (кап 500 записей), rich-таблица с
  трендом и самодостаточный HTML (line-chart total_diffs по таблицам,
  вердикт «ДРЕЙФ НУЛЕВОЙ — можно переключаться»)
- **feat(web)**: `dbparity serve` — локальная веб-консоль (stdlib-only):
  запуск сверок из браузера, live-прогресс поллингом, раздача HTML/JSON
  отчётов; только localhost, отчёты строго в workdir консоли
- **community**: CONTRIBUTING, SECURITY, issue-формы (bug, feature,
  field-report для реальных Oracle/MSSQL прогонов), PR-шаблон
- **test(mssql)**: самодиагностирующиеся assert'ы hash-теста (причина
  видна прямо в сообщении CI)
- **.gitignore**: рабочие файлы прогонов (чекпоинты, инкремент-стейты,
  каталог консоли)

## 0.6.0 — 2026-07-03

Три независимых блока (параллельная разработка).

- **feat(mssql)**: полноценный адаптер — ODBC Driver 18 (dsn или
  host/port/…), конвертер datetimeoffset, digest-API с T-SQL канонизацией
  чисел (совместима с trim_scale/TM9), бинарная коллация; live-джоба в CI
  на mcr.microsoft.com/mssql/server:2022 + env-gated интеграционные тесты
- **feat(core)**: инкрементальный режим для dual-write — `incremental:
  {таблица: wm_колонка}`: после полного прогона сверяются только изменённые
  строки; missing/extra среди них = дрейф; стейт-файл с fingerprint
  конфига, `--full` для полной пересверки; несовместимые режимы
  (hash/resume) разрешаются автоматически с заметкой в отчёте
- **feat(report)**: формат JSON-отчёта заморожен — `schema_version: 1`,
  правила эволюции, справочники docs/report-format.md и
  docs/config-reference.md, золотой тест схемы в CI

## 0.5.0 — 2026-07-03

Три независимых улучшения (параллельная разработка).

- **feat(adapters)**: бинарные коллации для текстовых PK — `COLLATE BINARY`
  (sqlite) / `COLLATE "C"` (PG) / `NLSSORT BINARY` (Oracle) /
  `Latin1_General_BIN2` (MSSQL); порядок merge больше не зависит от
  локалей движков, предупреждение заменено гарантией
- **feat(cli)**: `dbparity validate -c config.yaml` — проверка конфига без
  подключения к БД: обязательные поля по типу эндпоинта, типы значений,
  подсказки при опечатках; `compare` теперь показывает все ошибки конфига
  разом
- **infra**: бенчмарк-матрица в CI — `bench --json`, workflow с порогами
  регрессии производительности и метриками в summary

## 0.4.0 — 2026-07-03

Resilience: обрывы больше не обнуляют многочасовую сверку.

- **feat**: checkpoint/resume — атомарный JSON-стейт с fingerprint конфига;
  завершённые таблицы восстанавливаются без пересчёта, прерванная
  продолжается с PK-watermark (`checkpoint:` в конфиге, `--resume` в CLI).
  Партиал-слот на таблицу: упавшая таблица не теряет стейт из-за соседей
- **feat**: ретраи на ошибки БД/сети — `retry_attempts`/`retry_backoff_s`,
  свежая пара соединений на каждую попытку, продолжение с watermark
  внутри того же запуска
- **feat**: open-ended `pk_range` (WHERE pk >= lo) во всех адаптерах
- **refactor(engine)**: соединения выдаются на таблицу (а не на прогон) —
  чище ретраи, дружелюбнее к пулерам и однопоточным окружениям
- Паритет с эталоном при обрыве закреплён тестами (симуляция обрыва
  сети на середине потока, точность счётчиков строк на границе watermark)

## 0.3.0 — 2026-07-02

Big tables: hash-режим.

- **feat**: бакетные DB-side хэши за один скан (`strategy: auto|stream|hash`,
  `hash_leaf_rows`) — совпавшие PK-бакеты зачитываются агрегатами без
  передачи строк; расходящиеся детализируются потоковым merge с полной
  нормализацией. На бенче 300K×2 с 3 диффами передаётся 60K строк вместо 600K
- **feat**: digest-API адаптеров (sqlite: Python-UDF md5; PG: md5+trim_scale;
  Oracle: STANDARD_HASH+TM9, experimental); кросс-движковая канонизация
  чисел (`100.00` == `100` == `100.0`) проверена live-тестом sqlite↔PostgreSQL
- **feat**: pk_range-фильтр в stream_rows всех адаптеров
- **fix(postgres)**: опция `prepare_threshold` (окружения с общей сессией:
  PGlite, пулеры в transaction-режиме)
- Паритет hash vs stream закреплён тестами; NULL-PK и пустые таблицы
  корректно обрабатываются в hash-режиме

## 0.1.0 — 2026-07-02

Первый публичный релиз (alpha).

- Ядро: стриминговое merge-сравнение по PK, O(n)/O(batch), fast-path
  по логическим типам колонок (~400K строк/с на 1М строк)
- Нормализация «ловушек миграций»: Oracle `''`==NULL, хвостовые нули,
  epsilon для float, таймзоны, CHAR-паддинг, юникод NFC, `0/1/Y/N`↔bool,
  BLOB→MD5, точность timestamp
- Сверка схемы: колонки, логические типы, PK (регистронезависимо)
- Адаптеры: SQLite (+эмуляция диалектов), PostgreSQL (psycopg3,
  server-side курсоры; проверен на живом PG 18), Oracle (thin,
  experimental — нужны тестеры), MSSQL (каркас)
- Отчёты: самодостаточный HTML (Tabler + Chart.js, тёмная тема,
  маскирование значений) и JSON; CLI с кодами выхода для CI/CD
- 34 автотеста, включая live-PostgreSQL интеграцию
